# 安全治理与最佳实践

这一页适合作为 `07-安全与治理` 的总览入口。重点不是把所有风险都写成抽象原则，而是把科研自动化系统最常见的高风险点、公开规范和落地做法放到同一页，便于从“会调用工具”推进到“调用过程可控、可审计、可复核”。

## 二、科研自动化系统最常见的高风险点

| 风险点 | 典型场景 | 更稳妥的做法 |
| --- | --- | --- |
| 间接 prompt injection | 网页、PDF、issue、邮件、补充材料中夹带恶意指令。 | 把外部内容视为不可信输入；对网页抓取、RAG、PDF 解析和代码仓库摘要建立隔离层。 |
| 工具越权与副作用扩散 | agent 直接写仓库、发邮件、调用外部 API、提交评审意见。 | 默认最小权限；把写操作与对外操作放进人工复核或审批门。 |
| 证据链断裂 | 综述、novelty 判断或 rebuttal 只保留二手摘要。 | 关键 claim 必须保留 DOI / arXiv / OpenReview / OpenAlex 等原始来源。 |
| 实验不可复验 | 只保留最终图表，没有参数、环境、日志和 artifact。 | 保留配置、依赖、运行环境、日志、模型版本和数据版本。 |
| 敏感数据与保密材料外泄 | 未公开论文、评审稿、IRB 数据、合作项目材料进入外部工具。 | 先做数据分级；高敏材料优先在本地或受控环境内处理。 |
| 同行评审保密义务破坏 | 评审稿全文、审稿意见或评审讨论上传到第三方系统。 | 以 venue 规则、COPE / ICMJE 指导与保密要求为先。 |
| 软件与模型供应链风险 | 下载不明镜像、依赖漂移、模型权重或脚本来源不明。 | 使用签名、SBOM、provenance、依赖锁定和镜像扫描。 |

## 三、五条最值得先固化的治理原则

### 1. 文献、评审与结论必须可追溯

- 关键结论应能回到原始论文、原始评审意见、原始数据或原始实验日志。
- 自动化摘要、翻译、综述和 review draft 只应作为中间材料，不应替代原始证据。
- 对外发布前，应能回答“这段话来自哪篇文献、哪次实验、哪份审稿记录”。

### 2. 工具能力必须默认最小权限

- 外部检索、只读仓库访问、只读文献库访问，通常适合作为默认开放能力。
- 写仓库、执行 shell、访问生产数据库、调用投稿系统、发送邮件或消息，通常应转入审批模式。
- 远程 `MCP` 与 API 应尽量使用 scoped credential、短期令牌与项目级隔离。

### 3. 有副作用的步骤必须设置人工 gate

- 适合固定设置人工复核的环节包括：研究方向 freeze、novelty 结论、关键实验结论、审稿意见、response letter、正式投稿包、对外演示与 release notes。
- 带副作用的自动化链路越长，对中间态日志、操作回放和撤销机制的要求越高。

### 4. 高风险数据必须按分级处理

- 人体研究、医疗、生物、企业合作、未公开论文、评审稿和保密材料，不应与普通公开网页抓取或开放平台处理混在同一条流水线中。
- 数据分级、脱敏、最小保留周期、项目边界和访问审计应先于模型调用设计。

### 5. 评测与交付必须保留 provenance

- 研究过程至少应能回溯到数据来源、代码版本、配置、依赖、模型权重版本和运行环境。
- 对发布包、镜像、容器、模型和自动生成文稿，优先引入签名、SBOM、构建来源证明和发布记录。

## 四、分阶段治理检查点

| 阶段 | 建议检查点 | 关注重点 |
| --- | --- | --- |
| 课题扫描与选题 | `topic freeze` | 文献来源是否可靠，是否误把二手总结当成原始证据。 |
| 新颖性判断与综述 | `novelty gate` | novelty 结论是否保留原文比对、检索日志与排除理由。 |
| 实验执行与复现 | `experiment gate` | 是否保留配置、环境、日志、artifact 与异常记录。 |
| 评测与红队 | `safety eval gate` | 是否做过注入、越权、有害输出和工具滥用测试。 |
| 写作与投稿 | `submission gate` | authorship、引用、评审保密、AI 使用披露与模板检查是否完成。 |
| 交付与开源 | `release gate` | 是否附带 license、SBOM、签名、版本说明与 provenance。 |

## 五、工具层面的实际建议

| 工具类型 | 建议 |
| --- | --- |
| 学术检索类工具与 MCP | 默认把检索结果视为候选线索，而不是事实终稿；关键结论回到原始来源。 |
| 文献库、知识库、云盘类 MCP | 优先只读模式；最小化暴露目录、项目和文库范围。 |
| 网页抓取、浏览器、搜索类工具 | 对网页注入、广告脚本、恶意说明文本和不可见指令做额外防护。 |
| Shell、Notebook、Python 执行 | 放在隔离环境中运行，并记录输入、输出、文件变更和依赖快照。 |
| 写作、审稿、投稿工具 | 先区分公开材料与保密材料；对 venue 规则、AI disclosure 与 review confidentiality 做显式检查。 |
| 模型、依赖、容器与脚本下载 | 尽量选择官方源、锁定版本、补 SBOM 和镜像扫描，避免不明来源二进制。 |
