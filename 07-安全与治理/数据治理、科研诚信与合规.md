# 数据治理、科研诚信与合规

这一页聚焦科研工作流中最容易被忽略、但后果往往最严重的治理问题：研究不端、人体研究与敏感数据、同行评审保密义务、数据管理与共享、AI 使用披露、作者署名与许可证。

## 二、科研场景中的高风险材料

| 材料类型 | 典型风险 | 更稳妥的做法 |
| --- | --- | --- |
| 未公开论文与评审稿 | 泄露投稿内容、破坏评审保密义务。 | 先查 venue 规则、期刊政策和评审协议；必要时限制外部平台使用。 |
| 人体研究数据 | 违反 IRB、隐私保护与知情同意要求。 | 在研究开始前明确审批、脱敏、访问控制与保留周期。 |
| 企业合作或保密项目文档 | 违背保密协议、泄露商业秘密。 | 用项目边界隔离文档、模型与向量库；外发前走审批。 |
| 医疗、生物和专利材料 | 涉及隐私、监管或知识产权风险。 | 先确认数据来源、用途、再发布范围和法律约束。 |
| 审稿意见与 reviewer discussion | 泄露保密讨论或让外部系统接触评审内容。 | 以 venue 保密义务优先，外部工具仅处理经过授权的摘要或脱敏文本。 |

## 三、科研诚信与可披露性检查点

| 检查点 | 关注内容 |
| --- | --- |
| 文献与结论关系 | 关键 claim 是否能回到原始论文、数据或评审意见。 |
| AI 使用披露 | venue、期刊、资助方或机构是否要求披露 AI 的使用方式。 |
| 作者资格与贡献 | 是否把 AI 工具误写为作者，是否明晰人类贡献与责任。 |
| 数据来源与许可证 | 数据、图表、模型与代码是否有明确来源和合法再利用边界。 |
| 复现材料 | 是否保留方法、环境、版本、配置、脚本与补充材料。 |
| 审稿保密 | 是否把评审材料泄露到不适当的外部系统或公开仓库。 |

## 五、数据管理与共享的常见落地做法

| 主题 | 推荐做法 |
| --- | --- |
| 数据管理计划 | 对数据来源、格式、责任人、共享策略、长期存储与例外情况做前置说明。 |
| 数据分级 | 至少区分公开、项目内受限、敏感、保密四类。 |
| 访问控制 | 对文献库、网盘、数据库、向量库与实验平台做项目级隔离。 |
| 脱敏与最小保留 | 高敏数据优先脱敏；仅保留完成研究所需最小集。 |
| 元数据与引用 | 为数据、代码、文稿和补充材料保留 DOI、版本号、许可证和贡献记录。 |
| 公开共享 | 在法律、伦理和协议允许的前提下，优先补齐 README、license、metadata 与 data card。 |

## 七、最常用的检查清单

1. 当前材料是否包含未公开论文、评审稿、敏感数据或保密项目内容。
2. 当前任务是否触发 IRB、隐私、版权、许可或数据共享限制。
3. 关键结论是否仍能回到原始文献、原始数据或原始评审记录。
4. 是否明确了 AI 工具在写作、翻译、整理、审稿或分析中的使用边界。
5. authorship、contributorship、acknowledgement 与 disclosure 是否符合 venue 或期刊要求。
6. 数据、图表、代码、模型和文稿是否附带来源、版本与许可证信息。
