## 3. 建议的实验工程最小闭环

为了让科研系统真正辅助实验，建议至少保留下列 artifact：

1. `exp_plan.yaml`
2. `configs/`
3. `run_logs/`
4. `results.json`
5. `ablation.csv`
6. `environment lock`

这些东西比“聊天记录里说自己跑过实验”重要得多。

## 4. 一个常见误区

很多研究系统会在“生成代码”这一步表现得很厉害，但真正卡住人的往往是：

- 环境不一致
- 数据版本不清
- 配置没留档
- 指标和表格对不上
- 结果无法重跑

所以在实验层，`DVC + MLflow + Hydra + Jupyter 执行层` 的价值往往比再多一个聊天 agent 更高。
