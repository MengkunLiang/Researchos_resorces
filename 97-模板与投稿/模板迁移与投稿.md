# 模板迁移与投稿

这部分的重点不是“排版工具大全”，而是“哪些工具最值得接进科研智能体的投稿末端”。

## 1. 最关键的投稿资源

| 资源 | 场景 | 简述 | 链接 |
| --- | --- | --- | --- |
| Auto-Resubmit | 转投模板迁移 | 能在 ACL、EMNLP、NeurIPS、ICML、ICLR、CVPR、ICCV、AAAI 等模板家族间做无损迁移。 | [LilanOvO/Auto-Resubmit](https://github.com/LilanOvO/Auto-Resubmit) |
| OpenDraft | 长文档整理 | 对论文成稿、章节组织、输出多个格式比较友好。 | [federicodeponte/academic-thesis-ai](https://github.com/federicodeponte/academic-thesis-ai) |
| Open Research | reviewer-response | 对 rebuttal、response letter 和 methodology critique 很有帮助。 | [open-research.info](https://www.open-research.info/) |

| 资源 | 场景 | 简述 | 链接 |

## 3. 一条更稳的投稿链路

推荐做法：

1. `PaperQA2 / paper-glance-skill` 保证内容证据链
2. `OpenDraft / Anthropic Skills` 组织文本与图表说明
3. `Open Research` 生成 reviewer response 草稿
4. `Auto-Resubmit` 处理模板迁移
5. `Tectonic / latexmk` 负责编译检查

## 4. 什么时候必须人工把关

下面这些节点，不建议完全自动化：

- 结论定稿
- novelty 表述
- 引用正确性
- 投稿 venue 选择
- 最终提交文件检查

换句话说，投稿链路适合“高度辅助”，但不适合“完全放权”。
