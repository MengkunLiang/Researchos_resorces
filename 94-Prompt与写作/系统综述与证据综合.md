# 系统综述与证据综合

这一页聚焦 literature review、rapid review、systematic review、research report 与 evidence synthesis 写作。相比一般学术写作工具，这一类资源更强调检索、筛选、数据抽取、报告生成与可回查证据。

## 1. 什么时候进入这一页

- 需要把多篇论文组织成 related work、survey、rapid review 或 technical brief。
- 需要把检索、筛选、抽取和 synthesis 步骤做得更透明，而不是只生成一篇“看起来像综述”的文字。
- 需要把 review mining、novelty 判断和写作链路接回正式文稿。

## 5. 这类资源最适合产出哪些文稿

| 文稿类型 | 更适合优先参考的资源 |
| --- | --- |
| rapid review | `Consensus Pro Analysis`、`Elicit Reports`、`Open Research` |
| literature review / related work | `Elicit`、`STORM`、`GPT Researcher`、`RTI`、`survey-writer` |
| systematic review | `PRISMA 2020`、`Cochrane Handbook`、`Elicit Systematic Reviews`、`ASReview` |
| technical report / evidence brief | `Consensus`、`Elicit Reports`、`Open Research` |
| table-ready evidence summary | `benchmark-extractor`、`PaperQA2`、`OpenAlex MCP` |

## 6. 一条比较稳的证据综合链路

1. 用 `OpenAlex / Paper Search / OpenReview / Elicit / Consensus` 做 broad search 与 research question 澄清。
2. 用 `ASReview / litstudy / RTI / GPT Researcher / STORM` 做筛选、论文池整理与 evidence extraction。
3. 用 `benchmark-extractor / PaperQA2 / evidence card schema` 抽取对比维度和可回查证据。
4. 用 `survey-writer / ScholarCopilot / Open Research` 形成综述骨架、taxonomy 与 evidence brief。
5. 用 [学术写作与审稿](./学术写作与审稿.md) 里的写作与 rebuttal 工具完成后续成稿与修订。

## 7. 适合优先关注的几个评估点

| 评估点 | 说明 |
| --- | --- |
| 检索覆盖度 | 是否支持多轮检索、筛选与较大规模论文池。 |
| 流程透明度 | 是否保留 screening criteria、排除理由与中间步骤。 |
| 引用可回查 | 是否能回到具体论文、片段、页码或 supporting evidence。 |
| 输出可编辑性 | 是否允许修改 prompt、criteria、paper set 与输出结构。 |
| 导出能力 | 是否支持表格、报告、引用、过程记录或可继续写作的 skeleton。 |
