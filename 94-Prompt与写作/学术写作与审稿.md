# 学术写作与审稿

这一页聚焦“把研究材料组织成论文、技术报告、rebuttal、投稿稿件”的公开资源。重点不只是写得快，而是把 evidence、review、revision 和模板末端连起来。

## 1. 什么时候进入这一页

- 已经有笔记、图表、引用或实验结果，准备组织成 draft。
- 需要写 related work、response、rebuttal、cover letter 或 revision notes。
- 需要把 Prompt、Skill、MCP、模板与排版工具串成稳定写作链路。

## 6. 一条比较稳的写作与审稿链路

| 阶段 | 适合优先参考的资源 |
| --- | --- |
| 证据抽取与笔记 | `PaperQA2`、`paper-glance-skill`、`GPT Academic` |
| related work 与 draft skeleton | `survey-writer`、`OpenDraft`、`ScholarCopilot`、`Paperpal` |
| critique 与 rebuttal | `academic-paper-reviewer`、`review-rebuttal`、`Open Research`、`OpenReview` |
| 模板、协作与投稿 | `Zotero MCP`、`OverleafMCP`、`Quarto / MyST / Manubot`、`Auto-Resubmit` |

## 7. 完成这一页对应任务时，最低应该拿到什么

1. 一份结构稳定的 draft 或 report skeleton。
2. 一份可回查引用和 evidence 的 supporting structure。
3. 一份 reviewer concern / rebuttal / response 的表格化产物。
4. 一个可进入模板迁移与投稿末端的工程出口。
