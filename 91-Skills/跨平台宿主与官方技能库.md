# 跨平台宿主与官方技能库

这一页专门回答一个很容易被忽视的问题：

科研团队收集 Skills，不能只盯着某一个客户端。真正高价值的是那些既有开放标准、又有官方维护技能库、还能跨 `Codex / Claude Code / Copilot / Cursor / Gemini / OpenHands` 迁移的生态。

## 1. 先看最关键的官方与准官方入口

| 资源 | 角色 | 简述 | 链接 |
| --- | --- | --- | --- |
| Agent Skills 标准 | 开放规范 | 定义 `SKILL.md`、目录结构、脚本与资源组织方式，是整个生态的根。 | [agentskills.io](https://agentskills.io/) |
| Agent Skills 规范与参考 SDK | 规范实现 | `agentskills/agentskills` 提供规范、文档和参考实现，适合理解什么叫“标准兼容 skill”。 | [agentskills/agentskills](https://github.com/agentskills/agentskills) |

| Anthropic Skills | Claude 官方技能库 | 文档处理、设计、MCP、skill 构建等高频能力最成熟。 | [anthropics/skills](https://github.com/anthropics/skills) |

## 2. 不同宿主对 Skills 的支持有什么现实意义

| 宿主 / 生态 | 常见路径 | 现实意义 | 参考 |
| --- | --- | --- | --- |
| Codex | `.codex/skills/` | 如果团队有人用 Codex 做代码、文档、实验自动化，`openai/skills` 非常值得长期跟。 | [openai/skills](https://github.com/openai/skills) |
| Claude Code | `.claude/skills/` | 当前科研社区最活跃的 Skills 生态之一，科研写作和科学计算资源很多。 | [Claude Code Docs](https://docs.claude.com/en/docs/claude-code/skills) |
| GitHub Copilot | `.github/skills/` / `.claude/skills/` / `.agents/skills/` | 对团队仓库内共享技能很方便，适合把实验室方法论沉淀到 repo。 | [GitHub Docs](https://docs.github.com/en/copilot/concepts/agents/about-agent-skills) |
| Cursor | `.cursor/skills/` | 很多社区 skills 已经明确兼容 Cursor，适合做研究编码和快速验证。 | [skills.sh](https://skills.sh/) |
| Gemini CLI / Antigravity | `.gemini/skills/` / `.agent/skills/` | 适合做跨工具实验，避免把技能绑死在单一厂商。 | [VoltAgent/awesome-agent-skills](https://github.com/VoltAgent/awesome-agent-skills) |
| OpenHands | 公共 skills registry | 适合把 skills 当成平台能力层，而不仅是本地插件。 | [OpenHands Docs](https://docs.openhands.dev/usage/prompting/microagents-public) |

## 5. 面向科研资源库的理解方式

不要把 Skills 看成“某个 AI 客户端的插件市场”，而要看成三层结构：

1. 标准层：`Agent Skills`
2. 宿主层：`Codex / Claude Code / Copilot / Cursor / OpenHands`
3. 内容层：科研工程、写作审稿、科学计算、文档处理、知识库等具体 skill bundles

这样后续建设内部能力库时，就不会被单一产品绑定。

## 6. 建议优先跟踪的组合

1. 标准与规范：`agentskills/agentskills`
2. Codex 官方：`openai/skills`
3. Claude 官方：`anthropics/skills`
4. AI/ML 科研：`huggingface/skills`
5. 分发与发现：`vercel-labs/skills` + `skills.sh`

这五条线一起跟，基本就能把 Skills 生态的主干抓住。
