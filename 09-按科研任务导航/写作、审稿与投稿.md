# 写作、审稿与投稿

这一页适合在“已有笔记、证据、实验和图表，需要把它们转成投稿文本或返修材料”时使用。

## 一、什么时候进入这一页

- 已经形成较稳定的结果、图表和证据链。
- 需要写论文初稿、response、rebuttal 或 revision notes。
- 需要处理模板、Overleaf、LaTeX、preflight 和正式投稿系统。

## 四、最小可执行路径

1. 从知识库和实验页回收图表、表格、引用和证据卡。
2. 用 `Zotero MCP + citation-management` 固定 claim 与引用关系。
3. 用 `OpenDraft / ScholarCopilot / Overleaf / Quarto` 形成 grounded draft。
4. 用 `academic-paper-reviewer` 或 `AI Peer Review` 做内审后再准备投稿包。

## 五、推荐的成稿路径

1. 从 [知识管理与证据沉淀](./知识管理与证据沉淀.md) 回收证据卡和引用资产。
2. 用 `PaperQA2 + Zotero MCP + citation-management` 保证 claim 与证据一致。
3. 用 `OpenDraft / ScholarCopilot / Quarto / Overleaf` 形成 grounded draft。
4. 用 `academic-paper-reviewer` 或 `AI Peer Review` 做内审，再转 rebuttal / response。

## 六、完成这一步的最低产出

1. 一个结构完整的 draft。
2. 一份可回查引用和图表来源的证据链。
3. 一套 rebuttal / response / cover letter / revision notes 模板。
4. 一个可提交的投稿包或可复审的返修包。
