# 实验执行、复现与评测

这一页适合在“问题已经定下来，需要把 idea 落成实验闭环”时使用。

## 一、什么时候进入这一页

- 已经有明确研究问题和基线计划。
- 需要把 notebook、脚本、数据、环境和结果统一管理。
- 需要选 benchmark、做评测并保留完整留痕。

## 四、最小可执行路径

1. 用 `uv / conda-lock + Hydra` 固定环境与配置。
2. 选一组公开 benchmark 和一个可复现实验追踪栈。
3. 用 `Jupyter MCP / GitHub MCP + MLflow / DVC` 跑第一轮 baseline。
4. 产出可回放结果、artifact 和实验说明，再转入写作或回到 novelty 环节。

## 五、推荐的实验闭环

1. 先用 `Hydra + MLflow + DVC` 固定配置、追踪和数据版本。
2. 用 `Jupyter MCP / GitHub MCP` 搭可审计执行入口。
3. 根据方向选择 benchmark，必要时加 `Optuna` 做调参。
4. 在 [07-安全与治理](../07-安全与治理/README.md) 中补安全评测、日志与权限边界。

## 六、完成这一步的最低产出

1. 一条可重复运行的实验链。
2. 一套可回查的结果记录。
3. 一组公开或内部 benchmark 对照结果。
4. 一份足以写进论文的实验说明和 artifact 列表。
