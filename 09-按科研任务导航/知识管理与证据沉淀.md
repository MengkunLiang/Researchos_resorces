# 知识管理与证据沉淀

这一页适合在“已经有论文池、批注、Bib、项目笔记和实验材料，需要把它们沉淀成长期资产”时使用。

## 一、什么时候进入这一页

- 资料很多，但分散在 PDF、文件夹、浏览器收藏夹和聊天记录里。
- 已经有精读结果，但无法快速回到原始证据。
- 想让后续写作、审稿和复现实验都能直接复用已有材料。

## 二、先按这个顺序看

1. [03-知识管理与证据沉淀](../03-知识管理与证据沉淀/README.md)
2. [03-知识管理与证据沉淀/知识管理工具、MCP与知识载体详表](../03-知识管理与证据沉淀/知识管理工具、MCP与知识载体详表.md)
3. [95-文献检索与知识管理/知识管理与引用](../95-文献检索与知识管理/知识管理与引用.md)
4. [06-写作、审稿与投稿](../06-写作、审稿与投稿/README.md)

## 三、最小使用组合

| 资源类型 | 推荐先用 |
| --- | --- |
| 文献库 | `Zotero + Better BibTeX + Zotero MCP` |
| 知识库 | `Obsidian / Logseq` 或 `Notion MCP` |
| 代码与项目 | `GitHub MCP Server` |
| 证据问答 | `NotebookLM Skill`、`PaperQA2` 或 `Academic RAG MCP` |
| 写作出口 | `Manubot / Quarto / Zettlr` |

## 四、最小可执行路径

1. 把 PDF、Bib 和条目统一回收到 `Zotero` 或等价稳定文献库。
2. 把主题笔记、问题和对比表转入 `Obsidian / Logseq / Notion`。
3. 对关键 claim 保留 DOI、页码、引用语境和证据位置。
4. 用仓库、草稿和实验结果把这些资产接回 [写作、审稿与投稿](./写作、审稿与投稿.md) 或 [新颖性判断与综述](./新颖性判断与综述.md)。

## 五、推荐的沉淀方式

1. 文献条目和 PDF 先回到 `Zotero` 或等价稳定文献库。
2. 精读卡、主题笔记和研究问题放进 `Obsidian / Logseq / Notion`。
3. 对关键 claim 保留 DOI、页码、引文来源和证据截图位置。
4. 对实验与代码资产用 `GitHub + release / issue / notebook` 建立可回查连接。

## 六、完成这一步的最低产出

1. 一个稳定文献库。
2. 一套结构化笔记模板。
3. 一批证据卡片，而不只是自由文本笔记。
4. 与代码仓库、写作草稿和实验结果的明确连接点。
