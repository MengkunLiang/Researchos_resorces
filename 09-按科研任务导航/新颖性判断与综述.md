# 新颖性判断与综述

这一页适合在“已经有候选问题和论文池，需要判断 novelty、空白点与 reviewer concern”时使用。

## 一、什么时候进入这一页

- 想判断题目究竟是“足够新”，还是“已经很拥挤”。
- 需要把多篇论文的差异点、争议点和 reviewer concern 讲清楚。
- 需要做系统综述、evidence synthesis 或 review mining。

## 二、先按这个顺序看

1. [04-新颖性判断与综述](../04-新颖性判断与综述/README.md)
2. [04-新颖性判断与综述/新颖性判断、综述与评审工具详表](../04-新颖性判断与综述/新颖性判断、综述与评审工具详表.md)

4. [08-方向专题/科研新颖性与Idea生成](../08-方向专题/科研新颖性与Idea生成.md)

## 三、最小使用组合

| 资源类型 | 推荐先用 |
| --- | --- |
| Skills | [91-Skills/科研写作与文献Skills](../91-Skills/科研写作与文献Skills.md)、`peer-review`、`literature-review`、`scientific-critical-thinking`、`academic-paper-reviewer` |
| MCP | `OpenReview MCP`、`OpenAlex MCP`、`Paper Search MCP`、`Semantic Scholar MCP` |
| benchmark / 数据 | `llm4novelty`、`NLPeer`、`ReviewAdvisor`、`PeerRead`、`SciFact` |
| 工具 | `PaperQA2`、`scite`、`ASReview`、`Elicit / Consensus` |

## 四、最小可执行路径

1. 用 `OpenReview / OpenAlex / Paper Search` 建立相关工作池。
2. 用 `llm4novelty + NLPeer / ReviewAdvisor` 做 novelty 与 reviewer 语境判断。
3. 用 `PaperQA2 + scite + SciFact` 做 claim 与证据核验。
4. 输出“继续做 / 暂停 / 缩题 / 转方向”的明确决定。

## 五、推荐的判断路径

1. 先用 `OpenAlex / OpenReview / Paper Search` 建立相关工作池。
2. 用 `llm4novelty + NLPeer / ReviewAdvisor` 看 novelty 与 reviewer 语境。
3. 用 `PaperQA2 + scite + SciFact` 做 claim 与证据核验。
4. 如果题目仍然成立，再转向 [实验执行、复现与评测](./实验执行、复现与评测.md)。

## 六、完成这一步的最低产出

1. 一个清晰的 novelty 结论。
2. 一份 reviewer-style concern 列表。
3. 一份系统综述式差异图或 evidence map。
4. 一个继续做或停止做的明确决定。
