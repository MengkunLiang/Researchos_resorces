# 科研工作流设计模式

这一页不只列资源，而是把科研 Agent 系统中反复出现的模块与模式抽象出来，方便把 `Skills / MCP / Agent frameworks / knowledge assets / eval` 组合成可维护的工作流。

## 1. 常见工作流模式

| 模式 | 典型流程 | 适合关注的资源 |
| --- | --- | --- |
| 课题扫描与 related work | `query -> 搜索 -> 初筛 -> 主题聚类 -> 重点论文池` | [RTI](https://github.com/yrc-better/RTI)、[Tongyi DeepResearch](https://github.com/Alibaba-NLP/DeepResearch)、[Open Research](https://www.open-research.info/) |
| 单篇精读与证据卡片 | `PDF -> 结构化解析 -> QA -> citation grounding -> evidence card` | [PaperQA2](https://github.com/Future-House/paper-qa)、[STORM](https://github.com/stanford-oval/storm) |
| novelty 与审稿模拟 | `idea -> latest paper audit -> review simulation -> novelty note` | [llm4novelty](https://github.com/njust-winchy/llm4novelty)、[ReviewAdvisor](https://github.com/neulab/ReviewAdvisor)、[OpenReview](https://openreview.net/) |
| hypothesis generation | `observation/data -> hypothesis bank -> refinement -> ranking` | [HypoGeniC](https://github.com/ChicagoHAI/hypothesis-generation)、[AI Co-Scientist](https://research.google/blog/accelerating-scientific-breakthroughs-with-an-ai-co-scientist/)、[VirSci](https://aclanthology.org/2025.acl-long.1368/) |
| 实验设计与执行 | `experiment spec -> code plan -> run -> eval -> fix -> rerun` | [RD-Agent](https://github.com/microsoft/RD-Agent)、[OpenHands SDK](https://github.com/OpenHands/software-agent-sdk)、[AI-Scientist-v2](https://github.com/SakanaAI/AI-Scientist-v2) |
| 写作与审稿回复 | `artifact -> outline -> draft -> reviewer response -> resubmission` | [Open Research](https://www.open-research.info/)、[OpenDraft](https://github.com/federicodeponte/academic-thesis-ai)、[模板迁移与投稿](../07-模板与投稿/模板迁移与投稿.md) |
| 协作式持续研究 | `shared memory -> report exchange -> cumulative improvement` | [AgentRxiv](https://agentrxiv.github.io/)、[Zotero](https://github.com/zotero/zotero)、[OpenReview](https://openreview.net/) |

## 2. 一套常见的系统分层

| 层级 | 主要对象 | 代表资源 |
| --- | --- | --- |
| L1 数据与工具层 | 文献数据库、搜索、PDF、代码执行、实验环境 | `OpenAlex/OpenReview/Semantic Scholar/Zotero/Jupyter/SQL` 与对应 `MCP` |
| L2 方法层 | prompt、skills、固定流程模板、schema | `Skills`、`DSPy`、写作模板、审稿模板 |
| L3 编排层 | planner、executor、reviewer、router、state machine | `LangGraph`、`Microsoft Agent Framework`、`AutoGen`、`PydanticAI` |
| L4 资产层 | evidence card、paper note、hypothesis bank、run log、draft | `Zotero`、`MLflow`、`DVC`、本地知识库 |
| L5 评测治理层 | benchmark、trace、审计、人工 gate、权限控制 | `Aviary`、`LAB-Bench`、`MLE-bench`、权限与审批机制 |

## 3. 在科研系统里很实用的 artifact 设计

| artifact | 作用 |
| --- | --- |
| `query pack` | 保存课题描述、关键词、检索式、排除条件和目标会议/期刊。 |
| `paper card` | 保存摘要、贡献、方法、局限、可复现实验、引用片段。 |
| `evidence card` | 绑定结论与证据来源，方便后续写作和审稿回复。 |
| `hypothesis bank` | 保存候选想法、评分、来源依据、失败原因和迭代历史。 |
| `experiment spec` | 保存实验设置、模型、数据、指标、预算和环境要求。 |
| `run log` | 保存运行结果、错误日志、异常样本、消融与重试轨迹。 |
| `draft package` | 保存提纲、正文、图表、参考文献、rebuttal 要点。 |

## 4. 人工 gate 常见放置位置

1. 课题冻结前：确认任务边界、目标 venue、实验预算和伦理风险。
2. novelty 审计后：确认是否继续推进或更换方向。
3. 实验大规模运行前：确认代码、数据、评测脚本和资源配额。
4. 结果解释阶段：避免代理把噪声结果误判成研究结论。
5. 投稿输出前：检查引用、图表、claims、审稿回复措辞。

## 5. 一套较稳的 ResearchOS 组合方式

| 模块 | 适合组合的资源 |
| --- | --- |
| 文献与深搜 | `OpenAlex/OpenReview MCP` + `RTI` + `Tongyi DeepResearch` + `PaperQA2` |
| idea / novelty | `AI Co-Scientist` + `HypoGeniC` + `llm4novelty` + `ReviewAdvisor` |
| 执行与实验 | `OpenHands SDK` + `RD-Agent` + `Jupyter/SQL MCP` + `MLflow/DVC` |
| 写作与转投 | `Open Research` + `OpenDraft` + `模板迁移与投稿专题` |
| 评测与治理 | `Aviary` + `LAB-Bench` + `MLE-bench` + 人工 gate |
