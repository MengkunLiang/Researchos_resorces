# 学术搜索与开放图谱

这一页聚焦开放学术图谱、论文元数据 API、开放获取状态、作者标识与领域入口型平台。

## 1. 开放学术图谱与公共元数据

| 资源 | 类型 | 简述 | 链接 |
| --- | --- | --- | --- |
| OpenAlex | 开放学术图谱 | 适合主题 landscape、作者 / 机构分析、引文网络与概念演化研究。 | [OpenAlex](https://openalex.org/) |
| Semantic Scholar API | 学术检索 API | 可检索论文、作者、引用关系、摘要与相似文献。 | [Semantic Scholar API](https://www.semanticscholar.org/product/api) |
| Crossref REST API | DOI / 元数据 API | 适合 DOI 补齐、期刊元数据整理与出版信息追踪。 | [Crossref REST API](https://www.crossref.org/documentation/retrieve-metadata/rest-api/) |
| OpenCitations | 开放引文数据 | 提供开放引文索引、REST API 与 SPARQL 入口。 | [OpenCitations](https://opencitations.net/) |
| ORCID Public API | 作者标识 API | 适合处理研究者身份、公开 profile 与作者消歧。 | [ORCID Public API](https://info.orcid.org/what-is-orcid/services/public-api/) |
| Unpaywall API | 开放获取状态 | 适合由 DOI 查询开放获取版本与可访问全文线索。 | [Unpaywall API](https://unpaywall.org/api) |

## 2. 学科入口与平台型资源

| 资源 | 类型 | 简述 | 链接 |
| --- | --- | --- | --- |
| arXiv API | 预印本 API | 获取 arXiv 预印本元数据、检索结果与基础订阅能力。 | [arXiv API](https://info.arxiv.org/help/api/) |
| OpenReview | 论文与评审平台 | AI 顶会论文、评审、rebuttal 与 workshop 内容的重要入口。 | [OpenReview](https://openreview.net/) |
| Europe PMC | 生命科学文献 API | 生命科学、医学、预印本与全文检索的重要入口。 | [Europe PMC RESTful Web Service](https://europepmc.org/RestfulWebService) |
| CORE API | 开放获取全文 API | 汇聚大量开放获取论文的元数据与全文入口。 | [CORE API](https://core.ac.uk/services/api) |
| DBLP | 计算机学科索引 | 面向计算机学科的高质量开放书目数据库。 | [DBLP](https://dblp.org/) |
| Lens | 学术与专利联动平台 | 适合做学术文献与专利之间的跨域检索与分析。 | [Lens Scholarly Search](https://about.lens.org/scholarly-search-analysis/) |
| Papers with Code | 论文-代码-榜单平台 | 适合把论文、开源代码、任务与 benchmark 页面联动起来。 | [Papers with Code](https://paperswithcode.com/) |

## 3. 常见检索分工

| 场景 | 更适合先看什么 |
| --- | --- |
| AI/ML 最新进展 | `OpenReview + arXiv API + DBLP + Papers with Code` |
| 引文网络与作者机构分析 | `OpenAlex + Semantic Scholar API + OpenCitations` |
| DOI、期刊与出版元数据补齐 | `Crossref REST API + ORCID + Unpaywall` |
| 生物医药、生命科学与临床方向 | `Europe PMC + CORE + Lens` |
| 交叉创新、专利联动与技术扩散 | `Lens + OpenAlex + Crossref` |

## 4. 使用提示

- 同一篇论文在不同平台的元数据质量、更新速度与字段覆盖范围可能不同，交叉校验通常更稳。
- AI/ML 方向做 related work 时，会议版、arXiv 版与期刊扩展版经常并存，宜保留版本来源。
- 若后续需要知识图谱、RAG 或 novelty 分析，优先保留统一的唯一标识符与原始链接。
