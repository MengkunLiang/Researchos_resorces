# 研究情报平台与外部合集

这一页聚焦两个问题：

1. 哪里适合持续扫新，避免漏掉最新资源。
2. 哪些平台更适合做 topic scan、trend analysis、citation expansion 和 survey scouting。

## 一、研究情报平台

| 资源 | 类型 | 更适合做什么 | 链接 |
| --- | --- | --- | --- |
| OpenAlex | 开放学术图谱 | works、authors、institutions、topics、citation network 的宏观分析。 | [OpenAlex](https://openalex.org/) |
| Semantic Scholar | 学术搜索引擎 | 论文池初筛、author graph、citation graph。 | [Semantic Scholar](https://www.semanticscholar.org/) |
| OpenReview | 开放评审平台 | 顶会投稿、review、rebuttal 与最新热点追踪。 | [OpenReview](https://openreview.net/) |
| Connected Papers | 引文图谱平台 | 从 seed papers 扩展方法家族和相邻工作。 | [Connected Papers](https://www.connectedpapers.com/) |
| ResearchRabbit | 文献发现平台 | 适合长期跟踪一个方向的论文池演化。 | [ResearchRabbit](https://www.researchrabbit.ai/) |
| Litmaps | 文献地图平台 | 适合长期维护主题地图和引文网络。 | [Litmaps](https://www.litmaps.com/) |
| Open Knowledge Maps | 知识地图平台 | 适合 broad topic overview 和可视化知识地图。 | [Open Knowledge Maps](https://openknowledgemaps.org/) |
| scite | 引用语境平台 | 适合判断某项工作是被支持、被使用还是被质疑。 | [scite](https://scite.ai/) |
| Elicit | research assistant | 适合 broad search、question refinement 和 evidence synthesis。 | [Elicit](https://elicit.com/) |
| Consensus | research search | 适合把自然语言问题映射到论文证据和初步综合。 | [Consensus](https://consensus.app/) |

## 二、适合继续扫库的外部合集

| 资源 | 类型 | 为什么值得看 | 链接 |
| --- | --- | --- | --- |
| Awesome LLM Scientific Discovery | 科学发现合集 | 把 LLM in scientific discovery 按自治程度分成 Tool、Analyst、Scientist 三层。 | [HKUST-KnowComp/Awesome-LLM-Scientific-Discovery](https://github.com/HKUST-KnowComp/Awesome-LLM-Scientific-Discovery) |
| awesome-data-agents | data agents 合集 | 对 data discovery、analysis、modeling、benchmark 很有价值。 | [HKUSTDial/awesome-data-agents](https://github.com/HKUSTDial/awesome-data-agents) |
| awesome-ai-research-writing | 写作与技能合集 | 选题后续如果进入 survey、写作和审稿阶段，这一套资源很容易接上。 | [Leey21/awesome-ai-research-writing](https://github.com/Leey21/awesome-ai-research-writing) |
| littleben/awesomeAgentskills | Skills 合集 | 对找 community skills 和 research-adjacent skills 很有帮助。 | [littleben/awesomeAgentskills](https://github.com/littleben/awesomeAgentskills) |
| VoltAgent/awesome-agent-skills | Skills 合集 | 适合继续补跨宿主的 agent skills。 | [VoltAgent/awesome-agent-skills](https://github.com/VoltAgent/awesome-agent-skills) |
| TensorBlock/awesome-mcp-servers | MCP 合集 | 适合继续补学术检索、数据工程和知识库类 MCP。 | [TensorBlock/awesome-mcp-servers](https://github.com/TensorBlock/awesome-mcp-servers) |

## 三、几个特别值得接入选题模块的项目

| 资源 | 类型 | 作用 | 链接 |
| --- | --- | --- | --- |
| SurveyX | survey automation | 更适合在主题基本确定后把文献池转成 survey outline 和草稿。 | [IAAR-Shanghai/SurveyX](https://github.com/IAAR-Shanghai/SurveyX) |
| LitStudy | literature analysis | 适合在 Jupyter 中做文献池筛选、统计、网络分析和主题发现。 | [NLeSC/litstudy](https://github.com/NLeSC/litstudy) |
| Local Citation Network | 引文网络工具 | 适合围绕一个主题或一组 seed papers 构造局部 citation network。 | [LocalCitationNetwork](https://github.com/LocalCitationNetwork/LocalCitationNetwork.github.io) |
| OpenAlex API Tutorials | 教程集合 | 适合做 bibliometrics、topic analysis、institution analysis 的 notebook 参考。 | [ourresearch/openalex-api-tutorials](https://github.com/ourresearch/openalex-api-tutorials) |
| Open Knowledge Maps Headstart | 知识地图框架 | 适合理解如何把学术搜索结果变成知识地图。 | [OpenKnowledgeMaps/Headstart](https://github.com/OpenKnowledgeMaps/Headstart) |

## 五、建议使用方式

1. 想快速扫方向时，先用平台类工具。
2. 想继续补库时，再用外部合集。
3. 想把 topic scan 做成可复用流程时，转到 [选题用Skills、MCP与Agent详表](./选题用Skills、MCP与Agent详表.md)。
