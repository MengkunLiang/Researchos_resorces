## 二、按科研任务链路看

### 1. 课题扫描与 related work

- 多源检索：`paper-search-mcp`
- 顶会最新论文：`openreview/openreview-mcp`
- 学术图谱与 landscape：`hbiaou/openalex-mcp`
- 开放学术平台：`OpenAlex`、`Semantic Scholar API`、`Crossref`

### 2. 单篇论文精读与结构化加工

- 中文翻译与重编译：`arxiv-translator`
- 深度分析 / 审稿意见 / 导图：`paper-glance-skill`
- 论文问答与引用追踪：`PaperQA2`
- PDF 解析：`GROBID`、`Marker`、`Docling`

### 3. idea、novelty、风险审计

- 最新论文池：`openreview/openreview-mcp`
- 图谱和引文网络：`openalex-mcp`
- 研究工作台：`Open Research`
- novelty benchmark：`llm4novelty`、`ReviewAdvisor`、`NLPeer`
- hypothesis / idea：`AI Co-Scientist`、`HypoGeniC`、`ResearchAgent`、`VirSci`
- 闭环系统参考：`AI-Scientist-v2`、`Dolphin`

### 4. 实验实现与复现

- 实验技能库：`AI-research-SKILLs`、`HF Skills`
- notebook 交互：`jupyter-mcp-server`、`Jupyter AI`
- 数据与实验追踪：`DVC`、`MLflow`
- 配置管理：`Hydra`
- AI/ML benchmark：`OpenML`、`MLGym`、`MLE-bench`
- data mining 底座：`PyG / DGL / OGB / RecBole / PyOD / DoWhy`

### 5. 写作、审稿、转投

- Prompt 与教程：`awesome-ai-research-writing`、`HKUST-Got-Skills`
- 学术长文生成：`OpenDraft`
- LaTeX 模板迁移：`Auto-Resubmit`
- 文档编译与转换：`Typst`、`Pandoc`、`OverleafMCP`

### 6. 管理技术交叉 / computational design science

- 学术与创新数据：`OpenAlex`、`OpenReview`、`PatentsView`、`Lens`、`GDELT`、`OSF`
- 机制识别：`DoWhy`、`EconML`、`causal-learn`、`linearmodels`
- 行为实验：`oTree`、`jsPsych`、`lab.js`、`PsychoPy`

## 三、建议优先级

| 优先级 | 建议 |
| --- | --- |
| A | 优先接官方 / 高活跃资源，尤其是 `MCP` 和 `Skills` 的底座能力。 |
| A | 文献链路优先打通，因为它直接决定系统是否真正“懂研究现场”。 |
| B | 写作、转投和 reviewer-response 可以后接，但很适合快速体现价值。 |
| B | benchmark 不一定一开始就全做，但至少先留出 `MLE-bench / Aviary / LAB-Bench` 接口。 |
| C | 通用 agent 框架不要全收，选一个主编排框架即可。 |

## 四、第一版资源闭环

如果目标是先把仓库变成“能辅助真实科研”的版本，可先从下面这套组合开始：

1. 文献入口：`paper-search-mcp` + `openreview/openreview-mcp` + `openalex-mcp`
2. 单篇理解：`arxiv-translator` + `paper-glance-skill` + `PaperQA2`
3. 个人资产层：`Zotero` + `zotero-mcp` + `Better BibTeX`
4. 写作与转投：`OpenDraft` + `Auto-Resubmit`
5. 评测与治理：`MLE-bench` + `Aviary` + 人工 gate

若研究核心聚焦 `AI / ML / data mining + management / technology` 交叉，可在上面这条主链之外，再额外单独维护：

1. novelty 专题页
2. AI/ML/data mining 专题页
3. computational design science 专题页
